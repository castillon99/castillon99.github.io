Guía para empezar sin perderte

GitHub explicado como si fuera tu primer proyecto real.

Esta página es un tutorial práctico para entender qué puedes hacer con GitHub, por qué suele confundir al principio y cómo usarlo para crear páginas web, guardar código, documentar proyectos y trabajar con inteligencia artificial.

Idea central

GitHub no es solo una carpeta en internet. Es una bitácora de cambios, un respaldo de proyectos y una plataforma para publicar, revisar y colaborar.

proyecto/ ├─ index.html ├─ style.css ├─ README.md └─ historial de cambios ├─ commit 1: crear portada ├─ commit 2: ajustar diseño └─ commit 3: publicar en GitHub Pages
Investigación práctica

Complicaciones comunes al empezar

La mayoría de los principiantes no se atora porque GitHub sea imposible, sino porque mezcla varios conceptos nuevos al mismo tiempo: archivos, repositorios, ramas, commits, publicación web y sincronización.

1. Confundir Git con GitHub

Git es el sistema que guarda versiones. GitHub es la plataforma en internet donde subes y administras esas versiones.

2. No entender el repositorio

Un repositorio es la carpeta principal del proyecto. Ahí viven los archivos y el historial de cambios.

3. Miedo a romper algo

Los commits permiten regresar a versiones anteriores. La lógica no es “guardar perfecto”, sino avanzar con puntos de restauración.

4. Perderse con ramas

Una rama sirve para probar cambios sin tocar la versión principal. Al principio puedes trabajar en main, pero conviene aprender ramas pronto.

5. GitHub Pages no carga

Suele pasar por nombres incorrectos, archivos fuera de lugar, falta de index.html, mayúsculas/minúsculas o configuración de rama.

6. Subir archivos sin README

Sin README, el proyecto queda sin explicación. El README es la portada escrita del repositorio.

Ruta sugerida

Orden correcto para aprender GitHub

Regla de oro

Primero aprende GitHub desde la interfaz web. Después usa VS Code, GitHub Desktop o comandos. No intentes aprender todo al mismo tiempo.

Crear o abrir un repositorio

Piensa en él como la carpeta madre del proyecto. Para una página simple necesitas al menos un archivo index.html.

Editar archivos

Puedes crear archivos desde GitHub, subirlos manualmente o editarlos desde un programa como VS Code.

Hacer commits

Cada commit debe describir qué cambiaste: “Crear portada”, “Agregar sección de contacto”, “Corregir estilo móvil”.

Publicar con GitHub Pages

Activa Pages desde Settings → Pages. Normalmente se publica desde la rama main y la carpeta raíz o /docs.

Documentar con README

Explica qué hace el proyecto, cómo abrirlo, qué archivos contiene y qué cambios planeas hacer.

Herramientas que muchos no conocen

GitHub sirve para más que guardar código

README.md

Documento principal del proyecto. Sirve para explicar qué es, cómo se usa y qué falta por hacer.

GitHub Pages

Publica páginas web estáticas gratis desde un repositorio. Ideal para practicar HTML, CSS y JavaScript.

Issues

Lista de pendientes, errores o ideas. Puedes usarlos como mini tablero de trabajo.

Pull Requests

Propuesta formal de cambios. Permite revisar antes de unirlos a la rama principal.

Branches

Copias paralelas del proyecto para experimentar sin arriesgar la versión estable.

GitHub Desktop

Aplicación visual para sincronizar cambios sin usar comandos. Muy útil para principiantes.

VS Code

Editor de código. Permite abrir carpetas, editar archivos y conectarse con GitHub.

Actions

Automatizaciones. Más adelante puede servir para revisar, construir o publicar proyectos automáticamente.

Ejemplo práctico

Cómo se publica una página web simple

Una página web básica puede vivir en un solo archivo llamado index.html. GitHub Pages busca ese archivo para saber qué mostrar al visitante.

<!DOCTYPE html> <html lang="es"> <head> <meta charset="UTF-8" /> <title>Mi primera página</title> </head> <body> <h1>Hola, GitHub Pages</h1> <p>Esta página está publicada desde un repositorio.</p> </body> </html>

Errores típicos en Pages

El archivo se llama Index.html en vez de index.html; está dentro de una carpeta equivocada; Pages está apuntando a otra rama; el sitio tarda unos minutos en actualizar; o el navegador muestra caché vieja.

Práctica guiada

Mini proyecto recomendado

Construye una página de prueba

Objetivo: crear una página personal sencilla para aprender el flujo completo sin complicarte con frameworks.

  • Crear un repositorio.
  • Agregar index.html.
  • Agregar título y texto.
  • Hacer commit.
  • Activar GitHub Pages.
  • Crear README.
  • Agregar una sección nueva.
  • Hacer segundo commit.
  • Crear un Issue con una mejora pendiente.
  • Cerrar el Issue al terminar.

Consejo final

No empieces intentando crear una aplicación grande. Empieza con páginas pequeñas, cambios pequeños y commits frecuentes. GitHub se entiende mejor cuando lo usas como bitácora, no como examen técnico.
